Data Matching Algorithms

While tackling a big complicated problem with lots of intricacies (thousands of rows, one-to-many associations, different reporters sharing the same name) our team elected to break down our matching task into a filtration behavior. Smaller and faster algorithms find easy matches, filter down datasets, and pass remaining unmatched rows to the stronger but slower algorithms!

Cosine Matching

  • Extremely Fast
  • Passes through data first building the most obvious associations first
  • Works by converting strings to vectors, and compares similarity between the angles of various vectors

Fuzzy Matching

  • Extremely Fast
  • Analyzes data that Cosine couldn't match, applies fuzzy comparison logic to build more matches
  • Utilizes Levenshtein distance, which measures the number of edits needed to transform one string into another

Grouping Algorithm

  • Extremely Fast
  • Continues to filter down data by identifying matches that Cosine and Fuzzy didn't find
  • Uses a clustering algorithm to group unstructured data by identifying unique first name-last name pairs that map to a single firm, creating high-confidence matches.
  • For pairs mapping to multiple firms, assigns medium and low confidence groups by statistically imputing matches using the mode of firm associations, validated as accurate and effective.

Frequency Algorithm

  • Slow! But Mighty!
  • Analyzes the final sets of filtered data that has already been passed through the above three algorithms.
  • Builds a distribution of internal dictionaries mapping associations between Analysts, Contributors, and ESTIMID's
  • Constructs matches in a very slow process of reporting on respetive frequencies, resulting in a very high confidence of identified matches
